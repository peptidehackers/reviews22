# Retrieval Map

Use this map to choose the narrowest memory scope before searching broadly.

## Start Narrow

### User preferences or stable context

Read:

- `MEMORY.md`
- `USER.md`
- `TOOLS.md`

### Recent work or what changed today

Read:

- `memory/YYYY-MM-DD.md`
- yesterday's daily memory if needed

### Recurring project context

Read:

- `entities/projects/`

Start with:

- `entities/projects/openclaw-quality-system.md`
- `entities/projects/system-improvement.md`

### Governance, rules, or runtime behavior

Read:

- `entities/systems/openclaw-governance.md`
- `QUALITY-SYSTEM.md`
- `config/truth-policy.md`
- `config/boundary-contract.yaml`

### Historical architecture or old audits

Read:

- `openarc/`

Treat this as archive material unless current files confirm it.

## Search Order

1. curated memory
2. relevant entity page
3. recent daily memory
4. live code/config
5. archived analysis

## Conflict Rule

If files disagree:

- current verified code/config beats notes
- curated memory beats raw daily logs for current durable state
- archives are reference material, not automatic truth
