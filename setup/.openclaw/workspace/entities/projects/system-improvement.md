# System Improvement

## Objective

Make `.openclaw` a better quality system by importing useful operational patterns from strong agent repos without copying their excess scaffolding.

## Current Direction

- GStack influence: stronger workflow separation, review, investigation, QA, safety locks.
- GBrain influence: durable memory, read-before-act, write-after-learning, curated memory over raw logs.

## Constraints

- Prefer minimal mechanisms with real enforcement.
- Avoid dead ceremony and stale documentation.
- Verify changes directly when possible.
