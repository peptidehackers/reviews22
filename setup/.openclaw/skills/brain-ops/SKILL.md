# Brain Ops Skill

> Read before acting. Write after learning.

## Purpose

Make memory a real operational layer instead of an archive.

## When to Use

Use on any task where prior context could change the plan, the answer, or the risk.

That includes:

- recurring projects
- system changes
- user preferences
- follow-up work on earlier sessions
- questions about what changed, why, or how the system currently works

## Read Path

Before acting:

1. Read `workspace/RETRIEVAL-MAP.md` to pick the narrowest scope.
2. Read `workspace/MEMORY.md` in main sessions.
3. Read `workspace/entities/INDEX.md` and the relevant entity page if the task involves a recurring project, system, or person.
4. Read `workspace/memory/YYYY-MM-DD.md` for today and yesterday when recent chronology matters.
5. Apply `config/truth-policy.md` if prior notes conflict with current evidence.

## Write Path

After meaningful work:

1. Add the session fact to `workspace/memory/YYYY-MM-DD.md`.
2. Update `workspace/MEMORY.md` if the fact should survive beyond the day.
3. Update an entity page if the fact belongs to a recurring person, project, or system.

## Rules

- Curated memory beats raw logs.
- Scoped retrieval beats broad retrieval.
- Current verified state beats historical notes.
- If a past note is obsolete, mark it as superseded in curated memory or the entity page.
- Do not dump full transcripts into long-term memory.
