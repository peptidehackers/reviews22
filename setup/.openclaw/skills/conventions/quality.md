# Quality Convention

Quality is a loop, not a tone.

## Required Loop

1. Understand the request.
2. Bound the scope.
3. Search and read enough current evidence to prove the touch points.
4. Execute the smallest valid change.
5. Verify with evidence immediately after each mutation.
6. Report only what is proven.

## Standards

- Prefer small, testable changes over broad rewrites.
- Prefer direct evidence over plausible explanation.
- Prefer concise output over inflated output.
- Prefer one good verification step over five vague assurances.
- Prefer explicit recovery and learned lessons over repeated avoidable mistakes.

## Failure Mode to Avoid

The common bad pattern is:

`confident plan -> broad edit -> no verification -> polished summary`

Reject that pattern. Replace it with:

`bounded plan -> targeted discovery -> focused edit -> proof -> narrow claim`
